# PR Findings

Use this style for `/review:*` results.

- Start with a one-line verdict.
- Group findings under `P1`, `P2`, and `P3`.
- Each finding should name the file, explain the risk in one sentence, and give a concrete fix.
- After findings, include a short `Applied` section for fixes already made and a short `Next` section for anything left open.
- Keep the tone direct and reviewer-focused.
