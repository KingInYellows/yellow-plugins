# Linear Status Summary

Use this style for `/linear:*` responses.

- Lead with the issue identifier, title, and current status.
- Include PR linkage and workflow movement only if it changed or needs repair.
- Summarize recent comments or context briefly when they affect implementation.
- End with the next recommended action for the developer.
- Avoid repeating full issue descriptions unless the user asked for detail.
